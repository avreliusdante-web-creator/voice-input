# Чеклист подготовки к публикации

Перед загрузкой расширения в магазины проверьте следующее.

---

## 1. Манифест (manifest.json)

- [ ] **Формат:** Manifest V3 (Chrome, Edge, Opera). Для Firefox можно V2 или V3, для совместимости лучше V3.
- [ ] **Обязательные поля заполнены:**
  - `name` — название расширения
  - `version` — семантический номер (например, 1.0.0)
  - `description` — краткое описание для магазина
  - `icons` — 16, 48, 128 px (PNG)
  - `permissions` — только необходимые (у Voice input только `activeTab`; микрофон запрашивается в runtime через браузер)
- [ ] В пакет для загрузки не попали лишние файлы (docs, тесты, create-icons.html и т.п.).

**Текущее состояние Voice input:** манифест в формате V3, все поля заполнены, разрешения минимальны.

---

## 2. Политика конфиденциальности

- [ ] Создана страница с политикой конфиденциальности (даже если данные не собираются — лучше явно написать «мы не собираем никакие данные»).
- [ ] Страница доступна по постоянной публичной ссылке (ваш сайт, GitHub Pages или другой хостинг).
- [ ] В карточке расширения в каждом магазине указана ссылка на эту страницу.

**В проекте:** готовая страница **docs/privacy.html** (на английском и русском). Разместите её на GitHub Pages или своём сайте и укажите ссылку в магазине. Пример: `https://USERNAME.github.io/REPO/docs/privacy.html`.

---

## 3. Описания для магазинов

- [ ] **Подробное описание** подготовлено на **русском** и **английском** (для международных магазинов).
- [ ] В описании указано, что расширение **бесплатное**.
- [ ] Упомянута кнопка **«Поддержать разработчика»** (донат) — по желанию пользователя.

**В проекте:** готовые тексты в **docs/STORE_DESCRIPTIONS.md** — краткое описание (до 132 символов), подробное описание на EN и RU. Копируйте их в карточку расширения при публикации.

---

## Дополнительно

- **Скриншоты:** минимум 1, рекомендуется 3–5 (1280×800 или 640×400). Покажите панель на сайте, кнопку записи, вставку в поле ввода.
- **Объявление о правах (Chrome):** в разделе Permissions опишите простым языком, зачем нужны доступ к вкладке и микрофон (по запросу пользователя при нажатии «Запись»).

После прохождения чеклиста соберите ZIP (только manifest.json, content.js, content.css, папка icons/) и загружайте в магазин согласно **docs/PUBLISHING_GUIDE.md**.
