# Публикация и продажа расширения «Voice input» в магазинах браузеров

Краткое руководство: как упаковать расширение, где его публиковать, что нужно подготовить и как можно монетизировать.

---

## 0. Подготовка к публикации (общее)

Перед загрузкой в магазины убедитесь, что выполнены три пункта:

1. **Манифест (manifest.json)** — формат Manifest V3 для Chrome, Edge, Opera; все поля заполнены: name, version, description, icons, permissions (только необходимые). У Voice input только `activeTab`; микрофон запрашивается при нажатии «Запись» через API браузера.
2. **Политика конфиденциальности** — создана страница с текстом политики (даже если данные не собираются: «мы не собираем никакие данные»). Ссылку нужно указать в карточке расширения в каждом магазине. В проекте готовая страница: **docs/privacy.html** (EN + RU). Разместите её на GitHub Pages или своём сайте.
3. **Описание** — подробное описание на русском и английском для международных магазинов. Укажите, что расширение бесплатно и есть кнопка «Поддержать разработчика». Готовые тексты: **docs/STORE_DESCRIPTIONS.md**.

Подробный чеклист: **docs/PREPUBLISH_CHECKLIST.md**.

## 1. Упаковка расширения для публикации

### Что включить в архив

В пакет должны входить **только нужные для работы расширения файлы**:

- `manifest.json`
- `content.js`, `content.css`
- Папка `icons/` с иконками: **16×16**, **48×48**, **128×128** пикселей (PNG)

**Не включайте** в архив для магазина:

- Папки `docs/`, тесты, лишние скрипты
- Служебные файлы: `.git`, `README.md`, `USER_GUIDE.md`, `PUBLISHING_GUIDE.md`, `portfolio.html` (если не нужны в пакете)
- Исходники иконок (например, `create-icons.ps1`, `create-icons.html`), если они не используются расширением

### Иконки (обязательно для магазинов)

В `manifest.json` для Chrome Web Store и Edge Add-ons нужно указать иконки:

```json
"action": {
  "default_title": "Voice input",
  "default_icon": {
    "16": "icons/icon16.png",
    "48": "icons/icon48.png",
    "128": "icons/icon128.png"
  }
},
"icons": {
  "16": "icons/icon16.png",
  "48": "icons/icon48.png",
  "128": "icons/icon128.png"
}
```

Если у вас пока только скрипт/страница для генерации иконок — создайте PNG 16, 48 и 128 и положите в папку `icons/`.

### Как собрать ZIP для загрузки

1. Создайте папку, например `voice-notebook-package`.
2. Скопируйте в неё все перечисленные выше файлы и папку `icons/`.
3. Выделите **содержимое** папки (не саму папку), создайте ZIP-архив.
4. Имя файла может быть, например: `voice-notebook-v1.0.0.zip`.

**Важно:** в корне ZIP должны быть `manifest.json`, `content.js`, `content.css` и папка `icons/`, а не вложенная папка `voice-notebook-package/`.

Проверка: откройте `chrome://extensions` → «Загрузить распакованное расширение» → выберите распакованный вариант этой же папки и убедитесь, что расширение работает без ошибок.

## 2. Где публиковать

| Магазин | Браузер | Регистрация / стоимость | Комментарий |
|--------|---------|--------------------------|-------------|
| **Chrome Web Store** | Chrome | Разовый взнос **$5 USD** за аккаунт разработчика | Основной магазин для Chrome, большая аудитория |
| **Edge Add-ons** | Microsoft Edge | **Бесплатно** (аккаунт Microsoft) | Edge на базе Chromium, тот же формат расширений |
| **Firefox Add-ons** | Firefox | **Бесплатно** | Нужна отдельная сборка: у Firefox свои требования к манифесту и политикам; Web Speech API в Firefox ограничен |

Рекомендуемый порядок: сначала **Chrome Web Store** и **Edge Add-ons** (один пакет подходит для обоих). Firefox — отдельным шагом, если решите поддерживать.

## 3. Chrome Web Store — пошагово

### Регистрация

1. Перейдите на [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole).
2. Войдите в Google-аккаунт.
3. Оплатите **разовый регистрационный взнос $5 USD** (карта, в том числе виртуальная).
4. Примите соглашение разработчика.

### Подготовка карточки расширения

Перед отправкой на модерацию нужно заполнить:

- **Название** — например: «Voice input».
- **Краткое описание** (132 символа) — для списка расширений.
- **Подробное описание** — что делает расширение, как пользоваться, какие сайты поддерживаются. Можно взять текст из портфолио или README.
- **Категория** — например: Productivity («Продуктивность»).
- **Язык** — основной язык списка (например, русский).
- **Скриншоты** — минимум 1, рекомендуется 3–5 (1280×800 или 640×400). Покажите: popup, блокнот, панель на сайте.
- **Маленькая иконка** — 128×128 (уже есть в пакете).
- **Политика конфиденциальности** — обязательна, если расширение запрашивает права (микрофон, вкладки). Нужна публичная ссылка на страницу с текстом политики (хостинг на GitHub Pages, свой сайт или сервис для политик).

### Политика конфиденциальности

В проекте готовая страница **docs/privacy.html** (на английском и русском). В ней указано:

- расширение не собирает и не передаёт персональные данные;
- голос обрабатывается через API браузера (Chrome/Edge — сервисы Google);
- данные не сохраняются на стороне разработчика.

Разместите **docs/privacy.html** по постоянной ссылке (GitHub Pages, свой сайт) и укажите эту ссылку в карточке расширения в магазине.

### Загрузка и отправка на проверку

1. В Developer Dashboard: «Новый элемент» → загрузите ZIP с расширением.
2. Заполните все обязательные поля (описание, скриншоты, политика конфиденциальности, объявление о правах).
3. В разделе «Права» (Permissions) опишите, зачем нужны «Микрофон» и «Вкладки» (доступ к вкладкам) — понятным языком для пользователя.
4. Выберите тип публикации: «Публично» (всем) или «Только по ссылке» (для тестов).
5. Отправьте на проверку. Обычно рассмотрение занимает от нескольких дней до 1–2 недель.
6. После одобрения расширение можно опубликовать (или настроить отложенную публикацию после прохождения модерации).

Дополнительно: [Официальная документация по публикации в Chrome Web Store](https://developer.chrome.com/docs/webstore/publish).

## 4. Edge Add-ons — пошагово

### Регистрация

1. Зайдите в [Partner Center](https://partner.microsoft.com/dashboard) под учётной записью Microsoft.
2. Зарегистрируйтесь как разработчик приложений (если ещё не зарегистрированы) — для Edge Add-ons это бесплатно.

### Подготовка и загрузка

1. В Partner Center выберите создание расширения для Microsoft Edge.
2. Подготовьте описание, скриншоты, иконку (аналогично Chrome).
3. Загрузите **ZIP-файл** с расширением (тот же пакет, что и для Chrome).
4. Заполните метаданные и отправьте на сертификацию. После проверки расширение появится в [Edge Add-ons](https://microsoftedge.microsoft.com/addons).

Дополнительно: [Публикация расширения в Edge](https://learn.microsoft.com/en-us/microsoft-edge/extensions-chromium/publish/publish-extension).

## 5. Варианты монетизации (как начать «продавать»)

Расширение можно распространять бесплатно или монетизировать. Варианты:

### A. Бесплатное распространение

- Публикуете в Chrome Web Store и Edge Add-ons бесплатно.
- Подходит для портфолио, репутации, привлечения внимания к другим продуктам/услугам.

### B. Донаты и поддержка

- В описании или на странице расширения указываете ссылку: Donation Alerts, Boosty, Patreon, PayPal, «Сбер Спасибо» и т.п.
- В самом расширении можно добавить пункт меню или ссылку «Поддержать разработчика» (в popup или на странице блокнота).

### C. Платная версия в магазине

- **Chrome Web Store** поддерживает платные расширения и внутриприложение покупки через [Chrome Web Store Payments](https://developer.chrome.com/docs/webstore/payments).
- Нужна регистрация как продавца, настройка платежей (Google Pay и т.д.), соблюдение налоговых требований по странам.
- Вариант: бесплатная базовая версия + платное расширение функционала (например, дополнительные языки, офлайн-режим, если позже появится).

### D. Фримиум (бесплатно + премиум)

- Бесплатная версия в магазине: базовый набор (например, блокнот + панель на основных сайтах).
- Отдельный сайт/лендинг: «Премиум» — расширенная поддержка, дополнительные сайты, приоритетная помощь. Оплата через ваш сайт (карта, ЮKassa и т.д.), без встроенных покупок в магазине.

### E. Корпоративные / B2B

- Та же бесплатная или платная версия в магазине.
- Дополнительно: предложение компаниям (обучение, кастомизация, поддержка, лицензии). Обычно через сайт и контакты, не через магазин.

## 6. Чек-лист перед публикацией

- [ ] Иконки 16, 48, 128 px созданы и указаны в `manifest.json`.
- [ ] ZIP собран без лишних файлов, в корне — `manifest.json` и все ресурсы.
- [ ] Расширение проверено в Chrome и Edge в режиме «Распакованное расширение».
- [ ] Подготовлены скриншоты (минимум 1, лучше 3–5).
- [ ] Написаны краткое и подробное описание, указана категория.
- [ ] Опубликована политика конфиденциальности и ссылка добавлена в карточку.
- [ ] Для Chrome: оплачен разовый взнос $5 и заполнены все поля в Developer Dashboard.
- [ ] Решение по монетизации: бесплатно, донаты, платная версия или фримиум.
