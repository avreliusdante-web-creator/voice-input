# Лицензии и использование компонентов

Краткая проверка компонентов, на которых основано расширение «Voice input», и их совместимость с получением донатов.

---

## 1. Код расширения (ваш код)

- **Лицензия:** MIT (файл `LICENSE` в корне проекта).
- **Коммерческое использование и донаты:** разрешены. MIT явно разрешает использование «without restriction», в том числе коммерческое; получение донатов за распространение или поддержку проекта лицензией не запрещается.

---

## 2. Внешние библиотеки (npm и т.п.)

- **Используются:** нет. В проекте нет `package.json`, зависимости через npm не подключаются.
- **Вывод:** ограничений по лицензиям сторонних библиотек нет.

## 3. Шрифты

### DM Sans (упоминается в коде)

- **Где используется:** в `content.css` указан в `font-family: 'DM Sans', ...` как предпочтительный шрифт. Сами файлы шрифта расширением **не загружаются** — ни через Google Fonts, ни локально. Браузер подставляет DM Sans только если он уже есть в системе или на странице; иначе используются следующие в списке системные шрифты (`-apple-system`, `BlinkMacSystemFont`, `Segoe UI`, `sans-serif`).
- **Документация:** в `docs/portfolio.html` шрифт DM Sans подключается с Google Fonts — это только страница портфолио (документация), не часть кода расширения в смысле Chrome Extension.
- **Лицензия DM Sans:** SIL Open Font License 1.1 (OFL-1.1). Разрешает свободное использование, в том числе коммерческое; донаты и монетизация проекта OFL не запрещают.
- **Вывод:** использование названия шрифта в CSS и (при желании) подключение DM Sans через Google Fonts в документации не создаёт препятствий для донатов.

## 4. API браузера и платформы

### Web Speech API (SpeechRecognition)

- **Что это:** стандартный веб-API (спецификация W3C), встроенный в браузер. Не отдельная «библиотека» с собственной лицензией.
- **Использование:** вызовы `SpeechRecognition` / `webkitSpeechRecognition` в вашем коде. В Chrome/Edge распознавание обеспечивает сам браузер (сервисы Google и т.п.) по их правилам использования.
- **Коммерческое использование:** использование Web Speech API в коммерческих продуктах и расширениях распространено; сама спецификация не запрещает коммерческое использование или донаты.
- **Вывод:** лицензии, на которых «основан» плагин в части речи, здесь не ограничивают донаты.

### Chrome Extension API / Manifest V3

- **Что это:** платформа расширений Chrome/Edge и её API (манифест, content scripts, разрешения и т.д.).
- **Использование:** разработка и распространение расширения по правилам Chrome Web Store / Edge Add-ons. Правила магазинов касаются распространения и платежей в магазине, а не того, разрешено ли вам принимать донаты на стороне (сайт, ссылка в описании и т.д.).
- **Вывод:** использование платформы расширений само по себе не запрещает получать донаты.
