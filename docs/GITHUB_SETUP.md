# Выкладка расширения «Voice input» на GitHub

Пошаговая инструкция: подготовка проекта, инициализация Git, создание репозитория, первый пуш, GitHub Pages для политики конфиденциальности и релизы.

---

## Шаг 0. Что уже есть в проекте

| Компонент | Назначение |
|-----------|------------|
| **README.md** | Описание расширения, установка, использование, ссылки на документацию |
| **manifest.json** | Манифест расширения (Manifest V3) |
| **content.js**, **content.css** | Логика и стили: плавающая кнопка и панель на сайтах |
| **icons/** | Генератор иконок (create-icons.html) и при необходимости PNG 16/48/128 |
| **docs/** | Документация: портфолио, инструкция пользователя, публикация в магазинах, политика конфиденциальности, описания для магазинов, чеклист подготовки |
| **LICENSE** | Лицензия (MIT) |
| **.gitignore** | Исключение системных файлов, IDE, логов, при необходимости — иконок |

Расширение **не требует сборки**: после клонирования достаточно сгенерировать иконки (если не закоммичены) и загрузить папку в браузер.

## Шаг 1. Что коммитить: состав репозитория

### Обязательные файлы

| Файл или папка | Назначение |
|----------------|------------|
| `manifest.json` | Манифест расширения |
| `content.js` | Content script |
| `content.css` | Стили |
| `icons/create-icons.html` | Генератор иконок (16, 48, 128 px) |
| `icons/README.txt` | Подсказка по созданию иконок |
| `docs/*` | Вся документация (в т.ч. privacy.html, STORE_DESCRIPTIONS.md, PREPUBLISH_CHECKLIST.md) |
| `README.md` | Описание репозитория |
| `.gitignore` | Игнорируемые файлы |
| `LICENSE` | Лицензия MIT |

### Иконки (PNG 16, 48, 128)

- **Вариант А (рекомендуется):** Закоммитить готовые `icon16.png`, `icon48.png`, `icon128.png` в папку `icons/`. После клонирования расширение можно сразу установить в браузере.
- **Вариант Б:** Не коммитить PNG. В `.gitignore` раскомментировать строку `# icons/*.png`. В README указать: «Сначала создайте иконки через `icons/create-icons.html` и сохраните их в папку `icons/`».

### Что не коммитить

- Системные файлы (`.DS_Store`, `Thumbs.db` и т.п.) — уже в `.gitignore`.
- Папки IDE (`.idea/`, `.vscode/`) — в `.gitignore`.
- Готовые ZIP-архивы и папку `dist/` — в `.gitignore`.
- Документы/черновики не по проекту (например, `расширениявбраузер.docx`) — добавьте в `.gitignore` при необходимости: `*.docx` или конкретное имя файла.

Проверьте содержимое `.gitignore` в корне проекта; при варианте Б с иконками должна быть раскомментирована строка `icons/*.png`.

## Шаг 2. Инициализация Git в папке проекта

1. Откройте терминал (PowerShell или cmd) и перейдите в **корень проекта** (папка с `manifest.json`, например `ГолосвТекст` или `voice-input`):

   ```bash
   cd D:\ГолосвТекст
   ```
   (подставьте свой путь.)

2. Проверьте, есть ли уже репозиторий:

   ```bash
   git status
   ```
   - Если команда не найдена — установите [Git for Windows](https://git-scm.com/download/win) и повторите.
   - Если вывод «not a git repository» — инициализируйте:

   ```bash
   git init
   ```

3. Добавьте файлы и сделайте первый коммит:

   ```bash
   git add .
   git status
   git commit -m "Initial commit: Voice input — voice to text extension"
   ```

## Шаг 3. Создание репозитория на GitHub

1. Откройте [github.com](https://github.com) и войдите в аккаунт.

2. Справа вверху: **«+» → «New repository»**.

3. Заполните:
   - **Repository name:** например `voice-input` или `voice-notebook`.
   - **Description (необязательно):** например: «Browser extension: voice to text in social media and messengers».
   - **Public**.
   - **Не добавляйте** README, .gitignore или лицензию — они уже есть в вашем проекте.

4. Нажмите **«Create repository»**.

5. На странице нового репозитория GitHub покажет команды для подключения существующего локального репо. Они понадобятся на следующем шаге.

---

## Шаг 4. Подключение remote и отправка кода

В том же терминале в корне проекта выполните (подставьте свой логин и имя репозитория):

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПО.git
git branch -M main
git push -u origin main
```

- Замените `ВАШ_ЛОГИН` и `ИМЯ_РЕПО` на свои (например, `johndoe` и `voice-input`).
- При первом `git push` может потребоваться авторизация. Для HTTPS используйте [Personal Access Token (PAT)](https://github.com/settings/tokens) вместо пароля: **Settings → Developer settings → Personal access tokens → Generate new token**; при `git push` введите логин и в поле пароля — токен.

После успешного пуша код появится в репозитории на GitHub.

## Шаг 5. Настройка репозитория на GitHub

1. **About (описание и ссылка)**  
   В правой колонке страницы репозитория: **About → шестерёнка (Edit)**. Укажите короткое описание и при желании ссылку на сайт или портфолио.

2. **Ссылка на донат в GitHub**  
   Чтобы ссылка «Поддержать разработчика» была видна на главной странице репозитория:
   - Откройте **About → шестерёнка (Edit)**.
   - В поле **Website** вставьте URL страницы доната (Donation Alerts, Boosty, Patreon, Ko-fi и т.п.), например:  
     `https://www.donationalerts.com/r/ваш_ник`
   - Сохраните. Ссылка появится в блоке About справа и будет вести на вашу страницу приёма донатов.  
   Дополнительно можно упомянуть донат в **README.md** (например, в конце: «Если проект полезен, можно поддержать разработчика: [ссылка]»).

3. **Topics (теги)**  
   В том же блоке **About** добавьте теги, например:  
   `chrome-extension`, `edge-extension`, `speech-recognition`, `voice-to-text`, `webextension`, `browser-extension`.

4. **Лицензия**  
   Файл **LICENSE** (MIT) в корне проекта уже попадёт в репозиторий. На странице репо GitHub покажет его в блоке About.

5. **Секреты**  
   В коде расширения не должно быть API-ключей и паролей. Если позже настроите GitHub Actions — секреты храните в **Settings → Secrets and variables → Actions**.

## Шаг 6. GitHub Pages для политики конфиденциальности

Для публикации в магазинах нужна **постоянная ссылка на политику конфиденциальности**. Удобно раздавать её через GitHub Pages из папки `docs/` (там уже есть `privacy.html`).

1. В репозитории: **Settings → Pages** (в левом меню).

2. В разделе **Build and deployment**:
   - **Source:** Deploy from a branch.
   - **Branch:** выберите `main` (или `master`), папку — **/ (root)** или **/docs**.
   - Если выбран **root**: тогда по умолчанию будет открываться README; страница политики будет доступна по адресу:  
     `https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО/docs/privacy.html`
   - Если выбран **/docs** как корень Pages: главная может быть индексом в `docs/`; политика:  
     `https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО/privacy.html`

3. Сохраните настройки. Через 1–2 минуты страница станет доступна по указанному URL. Эту ссылку нужно указать в карточке расширения в Chrome Web Store / Edge Add-ons (см. **docs/PUBLISHING_GUIDE.md** и **docs/STORE_DESCRIPTIONS.md**).

## Шаг 7. Релизы (Releases) для пользователей

Чтобы пользователи могли скачивать готовую сборку расширения:

1. Соберите ZIP: в корне архива должны лежать `manifest.json`, `content.js`, `content.css` и папка `icons/` с иконками 16, 48, 128 px (см. **docs/PUBLISHING_GUIDE.md**).

2. В репозитории: **Releases → Create a new release**.

3. **Choose a tag:** создайте тег, например `v1.0.0`, и привяжите к ветке `main`.

4. **Release title:** например `Voice input v1.0.0`.

5. В описание можно вставить краткую инструкцию по установке (распаковать ZIP → Chrome/Edge → «Загрузить распакованное расширение»).

6. В **Attach binaries** приложите собранный ZIP (например, `voice-input-1.0.0.zip`).

7. Нажмите **Publish release**.

Ссылка на скачивание ZIP появится на странице релиза. Её можно давать в README или при ответах пользователям.

## Шаг 8. Дальнейшая работа с репозиторием

- **Обычные правки:**  
  `git add .` → `git commit -m "краткое описание изменений"` → `git push`.

- **Ветки и Pull Request:**  
  Для новой фичи: `git checkout -b feature/название` → коммиты → на GitHub создайте Pull Request из этой ветки в `main`.

- **Обновление релиза:**  
  Внесли правки → закоммитили и запушили → в **Releases** создайте новый тег (например, `v1.0.1`) и приложите новый ZIP.

- **Issues:**  
  Включены по умолчанию. Шаблоны: **Settings → General → Issues → Set up templates**.

## Клонирование и запуск (для тех, кто клонирует репо)

```bash
git clone https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПО.git
cd ИМЯ_РЕПО
```

- Если иконки **есть в репо** — откройте `chrome://extensions` (или `edge://extensions`), включите «Режим разработчика», «Загрузить распакованное расширение» и укажите папку клона.
- Если иконки **не закоммичены** — откройте в браузере `icons/create-icons.html`, нажмите «Download icons», сохраните PNG в папку `icons/`, затем загрузите расширение как выше.
